# 1. パスの理解

パスとは、ファイルやディレクトリの場所を表す文字列のこと

## 絶対パスと相対パス

| 種類 | 基準 | 例 |
| --- | --- | --- |
| 絶対パス | ルートディレクトリ `/` から始まる | `/home/user/projects/game.py` |
| 相対パス | 今いるディレクトリ(カレントディレクトリ)から始まる | `./projects/game.py` |

## 特別なディレクトリ記号

| 記号 | 意味 |
| --- | --- |
| `.` | カレントディレクトリ(今いる場所) |
| `..` | 親ディレクトリ(1つ上の階層) |
| `~` | ホームディレクトリ |
| `/` | ルートディレクトリ(最上位) |

## ディレクトリのツリー構造

```
/                        ← ルートディレクトリ
└── home/
    └── user/            ← ホームディレクトリ (~)
        └── projects/    ← カレントディレクトリ (.) とする
            ├── game.py
            └── images/
```

`projects/` にいるとき
- `./game.py` = `game.py`(カレントディレクトリのgame.py)
- `../` = `user/`(1つ上)
- `/home/user/projects/game.py` = 絶対パスで同じファイルを指す

# 2. コマンドでのディレクトリ操作

## 現在地を確認する

```bash
pwd
# Print Working Directory: 今いるディレクトリの絶対パスを表示
```

## ディレクトリの中身を見る

```bash
ls
# list: ファイルやディレクトリの一覧を表示

ls -l   # 詳細表示(Long)
ls -a   # 隠しファイルも表示(all)
```

## ディレクトリを移動する

```bash
cd ディレクトリ名   # change directory: 指定したディレクトリへ移動
cd ..              # 1つ上の階層へ移動
cd ~               # ホームディレクトリへ移動
cd -               # 直前にいたディレクトリへ移動
```

## ディレクトリを作成・削除する

```bash
mkdir ディレクトリ名          # make directory: ディレクトリを作成
mkdir -p 親/子/孫             # 上位ディレクトリも含めて一括作成

rmdir ディレクトリ名          # remove directory: 空のディレクトリを削除
rm -r ディレクトリ名          # ディレクトリと中身ごと削除
```

## ファイルを操作する

```bash
touch ファイル名              # 空のファイルを作成

rm ファイル名                 # ファイルを削除
rm -i ファイル名              # 削除前に確認する

cp コピー元 コピー先          # ファイルをコピー
mv 移動元 移動先              # ファイルを移動(ファイル名の変更にも使える)
```

## ファイルの内容を確認・書き込む

```bash
cat ファイル名                # ファイルの内容を表示する(concatenate)
cat -n ファイル名             # 行番号つきで表示する

echo 文字列                   # 文字列をターミナルに表示する
echo 文字列 > ファイル名      # 文字列をファイルに書き込む(上書き)
echo 文字列 >> ファイル名     # 文字列をファイルに追記する
```

## ターミナルを効率よく使う

```bash
# タブ補完: パスやコマンドを途中まで入力してTabキーを押すと自動補完される
cd Doc[Tab] → cd Documents/

# 上キー: 過去に入力したコマンドを1つずつさかのぼれる
↑キー  # 1つ前のコマンドを呼び出す
↑↑キー # 2つ前のコマンドを呼び出す

# which: コマンドの実体がどのパスにあるか確認する
which python   # → /usr/bin/python など
which python3  # → /usr/local/bin/python3 など
```

> `which python` でパスが返ってこない場合は `python3` を使う

# 3. Pythonの実行

## Pythonファイルを作成して実行する

```bash
# ステップ1. ファイルを作成する
touch game.py

# ステップ2. ファイルに内容を書き込む
echo 'print("Hello, World")' > game.py

# ステップ3. ファイルの内容を確認する
cat game.py

# ステップ4. Pythonで実行する
python ./game.py
```

`./` はカレントディレクトリを意味し、「今いる場所の `game.py`」を実行する

# 補足: テキストエディタ

`echo` では1行ずつしか書けないため、複数行を編集するにはエディタを使う

## vi (Visual Editor)

```bash
vi ファイル名      # viでファイルを開く
```

| 操作 | キー |
| --- | --- |
| 編集開始(入力モード) | `i` |
| 編集終了(コマンドモード) | `Esc` |
| 保存して終了 | `:wq` → Enter |
| 保存せずに終了 | `:q!` → Enter |

## nano

```bash
nano ファイル名    # nanoでファイルを開く
```

| 操作 | キー |
| --- | --- |
| 保存 | `Ctrl + O` → Enter |
| 終了 | `Ctrl + X` |
| 保存して終了 | `Ctrl + O` → Enter → `Ctrl + X` |

> nanoはviよりも直感的に操作できる。モードの切り替えが不要でキーボードに表示されるショートカットを見ながら使える
